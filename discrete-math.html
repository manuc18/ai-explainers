§1 — Propositional Logic

Propositions — The Atoms of Logic

A proposition is a declarative statement that is either true (T) or false (F) — never both, never neither. Propositions are the building blocks of all logical reasoning.

Definition

A proposition is a statement with a definite truth value. We use variables p, q, r, s to denote propositions.

▶ Quiz — Is It a Proposition?

Which of these is a proposition?

✓ "The square root of 4 is 2" is TRUE — it's a proposition. Questions and commands have no truth value. "x + 2 = 7" is a predicate (open statement) — its truth depends on x.

Is "This statement is false" a proposition?

✓ This is the Liar's Paradox. If true, it's false. If false, it's true. It cannot be assigned a consistent truth value, so it's NOT a proposition. Formal logic avoids self-referential statements.
§2 — Logical Operators

The Six Logical Connectives

New propositions are formed by combining simpler ones using logical connectives. Each has a precise definition via its truth table.

¬
Negation (NOT)
Flips the truth value. ¬p is true iff p is false.
∧
Conjunction (AND)
True only when both p and q are true.
∨
Disjunction (OR)
True when at least one of p, q is true.
→
Implication (IF→THEN)
False only when p is true and q is false.
↔
Biconditional (IFF)
True when p and q have the same value.
⊕
Exclusive OR (XOR)
True when exactly one of p, q is true.
The implication trap: p → q is only false when p is true and q is false. "If it rains, I carry an umbrella" is vacuously TRUE on sunny days — the premise is false, so the implication cannot be violated.
Precedence (high → low): ¬ > ∧ > ∨ > → > ↔
So: ¬p ∧ q → r parses as (¬p) ∧ q) → r

§4 — Logical Equivalence

Logical Equivalence & the Key Laws

Two formulas are logically equivalent (written p ≡ q) if they have the same truth value for every possible assignment of variables. Equivalence is the foundation of algebraic manipulation in logic.

§5 — Predicate Logic

Predicate Logic — Reasoning Over Domains

Propositional logic treats statements as indivisible atoms. Predicate logic (first-order logic) breaks them open — it lets us reason about the properties of objects and relationships between them.

Definition

A predicate P(x) is a function from a domain D to {T, F}. It becomes a proposition once x is assigned a value. The domain of discourse specifies the set of allowable values for variables.

P(x): "x is even" → P(4) = T, P(7) = F Q(x,y): "x > y" → Q(5,3) = T, Q(2,8) = F R(x): "x² = x" → R(0) = T, R(1) = T, R(2) = F
Key difference: P(x) alone is an open statement — not a proposition. It becomes one when you substitute a value OR when you bind x with a quantifier (∀x or ∃x).
§6 — Quantifiers

Universal & Existential Quantifiers

Universal Quantifier

∀x P(x) — "For all x in the domain, P(x) is true."
To disprove: find one counterexample. To prove: show it holds for every element.

Existential Quantifier

∃x P(x) — "There exists at least one x in the domain such that P(x) is true."
To prove: find one witness. To disprove: show it fails for all elements.

§7 — Nested Quantifiers

Nested Quantifiers — Order Matters!

When multiple quantifiers bind different variables, the order is critically important. Swapping them can completely change the meaning.

Domain: ℝ (real numbers), P(x,y): x + y = 0 ∀x ∀y P(x,y) → "For all x AND all y, x+y=0" FALSE ∃x ∃y P(x,y) → "Some x and some y exist where x+y=0" TRUE (e.g. x=1, y=-1) ∀x ∃y P(x,y) → "For every x, there's a y with x+y=0" TRUE (y = -x always works) ∃x ∀y P(x,y) → "One fixed x works for ALL y simultaneously" FALSE
∀x ∃y vs ∃y ∀x: In the first, y can depend on x. In the second, one y must work for all x simultaneously. The second is much stronger!
StatementReadingStrength

∀x ∃y P(x,y)For each x, some y worksStrong

§9 — Normal Forms

DNF and CNF — Canonical Forms

Every propositional formula can be rewritten in two canonical forms, useful for circuit design, SAT solving, and theorem proving.

Disjunctive Normal Form (DNF)

An OR of ANDs. Each AND clause (minterm) corresponds to one row of the truth table where the formula is TRUE.
Example: (p ∧ q) ∨ (¬p ∧ r) ∨ (q ∧ ¬r)

Conjunctive Normal Form (CNF)

An AND of ORs. Each OR clause (maxterm) corresponds to one row where the formula is FALSE, negated.
Example: (p ∨ ¬q) ∧ (¬p ∨ r) ∧ (q ∨ r)

§1 — Introduction to Proofs

What Is a Mathematical Proof?

A proof is a rigorous argument that establishes the truth of a mathematical statement beyond doubt. Every step must follow from definitions, axioms, or previously proven theorems.

Structure of a Proof

Theorem: A statement to be proved.
Hypothesis / Premises: What we are given (assume true).
Conclusion: What we must show follows from the hypothesis.
QED / □: Marks the end of the proof ("quod erat demonstrandum").

Strategy checklist: Before writing a proof, ask: (1) What are you given? (2) What must you show? (3) What definitions are relevant? (4) Try small examples first. (5) Choose the appropriate method.
§2 — Direct Proof

Direct Proof

The most straightforward method. To prove p → q: assume p is true, then use definitions, algebra, and known facts to derive q directly.

Template

1. Assume p (the hypothesis).
2. Apply definitions and known facts step by step.
3. Conclude q.

▶ Direct Proof — Step-Through
Theorem: If n is an even integer, then n² is even.
Step 1 — Assume hypothesis
Assume n is even.
This is what we are given.
Step 2 — Apply definition
By definition of even: n = 2k for some integer k.
Definition of even numbers.
Step 3 — Algebraic manipulation
Then n² = (2k)² = 4k² = 2(2k²).
Squaring both sides, factoring.
Step 4 — Apply definition again
Since 2k² is an integer, n² = 2(2k²) is divisible by 2.
Definition of even: 2 × integer.
Conclusion □
Therefore n² is even. ∎
We derived the conclusion directly.
§3 — Contrapositive Proof

Proof by Contrapositive

Instead of proving p → q directly, prove the logically equivalent ¬q → ¬p. Useful when working backwards from the conclusion is easier than forward from the hypothesis.

p → q ≡ ¬q → ¬p (these are always logically equivalent!)
▶ Contrapositive Proof — Step-Through
Theorem: If n² is odd, then n is odd.
Step 1 — Identify contrapositive
Instead prove: if n is NOT odd (even), then n² is NOT odd (even).
Contrapositive: ¬q → ¬p
Step 2 — Assume ¬q
Assume n is even. So n = 2k for some integer k.
Starting with the negated conclusion.
Step 3 — Derive ¬p
Then n² = (2k)² = 4k² = 2(2k²) — which is even.
Direct calculation.
Conclusion □
∴ If n is even → n² is even. Equivalently: if n² is odd → n is odd. ∎
Contrapositive established ≡ original theorem proved.
§4 — Proof by Contradiction

Reductio Ad Absurdum

Assume the negation of what you want to prove. Show this leads to a logical impossibility (contradiction). Therefore the original statement must be true.

Template

1. Assume ¬p (negation of what you want to prove).
2. Derive a contradiction: some statement C and ¬C both hold.
3. Since ¬p leads to impossibility, p must be true. □

▶ Classic Proof — √2 is Irrational
Theorem: √2 is irrational (cannot be expressed as p/q with integers p, q).
Step 1 — Assume the negation
Assume √2 IS rational. So √2 = p/q where p, q ∈ ℤ, q ≠ 0, and gcd(p,q) = 1 (fraction in lowest terms).
Setup for contradiction.
Step 2 — Algebraic manipulation
Squaring: 2 = p²/q² → p² = 2q²
Both sides squared.
Step 3 — First consequence
p² = 2q² means p² is even → p is even (by our previous theorem!).
So p = 2k for some integer k.
Contrapositive: n² even → n even.
Step 4 — Second consequence
Substituting: (2k)² = 2q² → 4k² = 2q² → q² = 2k²
So q² is even → q is even.
Same argument for q.
Step 5 — Contradiction! □
Both p and q are even → gcd(p,q) ≥ 2. But we assumed gcd(p,q) = 1. Contradiction!
∴ Our assumption was wrong: √2 is irrational. ∎
§5 — Proof by Cases

Proof by Cases (Exhaustion)

Divide the universe of possible inputs into exhaustive, mutually exclusive cases. Prove the theorem for each case. Together they cover all possibilities.

▶ Proof by Cases — Step-Through
Theorem: For any integer n, n² + n is even.
Setup
n is an integer. Every integer is either even or odd — these cases are exhaustive and mutually exclusive.
Case 1: n is even
n = 2k. Then n² + n = 4k² + 2k = 2(2k² + k). This is even. ✓
Definition of even numbers.
Case 2: n is odd
n = 2k+1. Then n² + n = (2k+1)² + (2k+1) = 4k²+4k+1+2k+1 = 4k²+6k+2 = 2(2k²+3k+1). Even! ✓
Algebra + definition of even.
Conclusion □
In both cases n² + n is even. Since every integer is even or odd, this covers all integers. ∎
§6 — Existence Proof

Existence Proofs

To prove ∃x P(x), you must show at least one element satisfying P(x) exists.

Constructive vs Non-Constructive

Constructive: Explicitly exhibit the witness. "Let x = 3; then x² − 1 = 8 = 2³ ✓"
Non-constructive: Prove existence without finding the specific witness, often via contradiction or pigeonhole.

▶ Non-Constructive Existence Proof
Theorem: There exist irrational numbers a, b such that aᵇ is rational.
Step 1 — Consider √2^√2
Let a = b = √2 (both irrational). Consider x = √2^√2.
x is either rational or irrational — two cases.
Case 1: x = √2^√2 is rational
Then a = b = √2 are irrational and aᵇ = √2^√2 is rational. Done!
Witnesses found: a = b = √2.
Case 2: x = √2^√2 is irrational
Let a = √2^√2 (irrational), b = √2 (irrational).
Then aᵇ = (√2^√2)^√2 = √2^(√2·√2) = √2² = 2. Rational!
Witnesses: a = √2^√2, b = √2.
Conclusion □
In either case, we've found irrational a,b with aᵇ rational. ∎
(Note: we don't know WHICH case holds — this is non-constructive!)
§7 — Uniqueness Proof

Uniqueness Proofs

A uniqueness proof has two parts: (1) show the object exists, (2) show it is the only such object.

Standard Template

1. Existence: Exhibit (or prove existence of) an object satisfying the property.
2. Uniqueness: Assume two objects x and y both satisfy the property, then prove x = y.

▶ Uniqueness of the Additive Identity in ℝ
Theorem: There is a unique real number e such that e + x = x for all x ∈ ℝ.
Existence
e = 0 satisfies 0 + x = x for all x ∈ ℝ. ✓
Verified by arithmetic axiom.
Assume two identities exist
Suppose e₁ and e₂ both satisfy: e₁ + x = x and e₂ + x = x for all x.
Uniqueness: assume two, prove they're equal.
Smart substitution
Apply e₁'s property with x = e₂: e₁ + e₂ = e₂
Apply e₂'s property with x = e₁: e₂ + e₁ = e₁
Conclusion □
Since addition is commutative: e₁ + e₂ = e₂ + e₁.
So e₂ = e₁. The additive identity is unique. ∎
§8 — Vacuous Proof

Vacuous and Trivial Proofs

Vacuous Proof

To prove p → q: if you can show p is FALSE, then p → q is TRUE regardless of q. (A false premise implies anything.)

Trivial Proof

To prove p → q: if you can show q is TRUE, then p → q is TRUE regardless of p. (An always-true conclusion makes any implication true.)

Vacuous Example: "All integers in the set {1, 2, 3} that are greater than 100 are prime." Since NO integer in {1,2,3} is > 100, the universal statement is vacuously true — there are no counterexamples.
▶ Quiz — Identify the Proof Type

"Prove: If 2 + 2 = 5, then the moon is made of cheese."

✓ Since "2+2=5" is false, ANY statement follows from it — this is vacuously true. This is why a false hypothesis "proves" everything: logical implication only fails when p is TRUE and q is FALSE.
§9 — Proof Strategy Guide

Choosing the Right Method

When should I use Direct Proof?
↓
When the hypothesis directly gives you enough information to algebraically or logically derive the conclusion. Start by writing "Assume p..." and see if you can reach q by applying definitions and known theorems.
When should I use Contrapositive?
↓
When the direct approach gets stuck, or when the negated conclusion gives more useful structure. Common signal: the conclusion is an existential ("there exists...") and the negation is a universal.
When should I use Contradiction?
↓
Ideal for proving something is IMPOSSIBLE or for showing uniqueness/non-existence. Classic uses: irrationality of √2, infinitely many primes, pigeonhole arguments. Assume the opposite, then find two contradictory facts.
When should I use Cases?
↓
When the domain naturally splits (even/odd, positive/negative/zero, or finitely many sub-cases). Make sure your cases are: (1) exhaustive — they cover everything, and (2) mutually exclusive — no overlap.
What's the most common beginner mistake?
↓
Assuming what you're trying to prove (circular reasoning). Also common: confusing the direction of a proof (proving p → q by assuming both p AND q). Always state clearly what you assume and what you're trying to show.

Sets — Collections of Distinct Objects

A set is an unordered collection of distinct objects called elements. Sets are fundamental — numbers, functions, relations, graphs are all defined in terms of sets.

§3 — Cardinality

Cardinality — Measuring the Size of Sets

The cardinality |A| is the "number of elements" in A. For finite sets this is straightforward. For infinite sets, we define it via bijections — a profound and surprising idea.

Definition (Equinumerosity)

Two sets A and B have the same cardinality (|A| = |B|) iff there exists a bijection (one-to-one correspondence) f: A → B. This definition works even for infinite sets!

Finite: |{a, b, c}| = 3 Infinite: |ℕ| = |{2n | n ∈ ℕ}| ← same size as all even numbers! Bijection: n ↦ 2n (maps every natural to a unique even) |ℕ| = |ℤ| = |ℚ| = ℵ₀ (countably infinite) |ℝ| = 𝔠 > ℵ₀ (uncountably infinite — proved below!)
§4 — Russell's Paradox

Russell's Paradox — Naive Set Theory Breaks

Bertrand Russell (1901) discovered a devastating contradiction in naive set theory (which allows any description to define a set).

▶ The Paradox — Step Through It
The Setup
Define R = {x | x ∉ x}. The "set of all sets that do not contain themselves."
Naive set theory: any predicate defines a set.
The Question
Does R ∈ R? (Is R a member of itself?)
Let's find out...
Case 1: Suppose R ∈ R
Then R satisfies the membership condition: R ∉ R. But we assumed R ∈ R. Contradiction!
R ∈ R → R ∉ R. ✗
Case 2: Suppose R ∉ R
Then R satisfies x ∉ x, so R should be IN R. But we assumed R ∉ R. Contradiction!
R ∉ R → R ∈ R. ✗
Conclusion — Naive Set Theory is Broken!
R ∈ R ↔ R ∉ R. No consistent truth value exists! Naive set theory is contradictory.
Resolution: ZFC Axiomatic Set Theory restricts which collections can be called "sets."
The Barber Paradox (Russell's popular version): "In a town, the barber shaves all and only those who do not shave themselves. Who shaves the barber?" — The same logical structure. The resolution: no such barber can exist.

§6 — Cantor's Diagonalization

Cantor's Diagonal Argument

Cantor's 1891 proof that ℝ is uncountable is one of the most elegant arguments in mathematics. It proves that no enumeration of real numbers can be complete — always missing infinitely many.

▶ Diagonal Argument — Animated

We represent real numbers in (0,1) as infinite binary sequences. Watch the argument unfold.

The key insight: the diagonal sequence, once flipped, differs from every row in at least one position. So it cannot be in any enumeration — no list can contain all reals.

Step 1 — Assume for contradiction
Suppose ℝ ∩ (0,1) is countable. Then we can list ALL binary sequences: r₁, r₂, r₃, ...
Step 2 — Extract the diagonal
Let dₙ = the n-th bit of rₙ (the diagonal). Form sequence d = d₁d₂d₃...
Step 3 — Flip each diagonal bit
Define s where sₙ = 1 if dₙ = 0, and sₙ = 0 if dₙ = 1. So s differs from d in every position.
Step 4 — The key observation
s ≠ r₁ (differs in position 1).
s ≠ r₂ (differs in position 2).
s ≠ rₙ for ALL n (differs in position n).
Contradiction □
s ∈ (0,1) but s is NOT in our list — contradicting the assumption that the list was complete. Therefore ℝ is uncountable. ∎
