Convolutional Neural Networks
from First Principles

How a sliding window of numbers learned to see edges, faces, and everything in between. From raw pixels to intelligent perception — built up piece by piece.

The core insight in one sentence: instead of memorizing every pixel arrangement, share a small filter across the entire image — and let the network learn what that filter should look for.
Section I

Introduction — Machines That See

For most of computing history, getting a machine to recognize a handwritten "7" from a "9" required painstaking hand-engineering: someone had to write rules describing every visual feature. This was fragile and didn't scale.

In 1989, Yann LeCun had a different idea. What if the machine could learn the features itself, directly from pixels? The result was the Convolutional Neural Network — a model so aligned with how visual data is structured that it dominated computer vision for three decades and still sits at the core of modern AI systems.

By the end of this article you'll understand every moving part: the convolution operation, feature maps, pooling, non-linearities, and how they stack into a complete learning system. Every concept has an interactive demo so you can feel it, not just read about it.

Section II

The Problem with Flat Networks

Before CNNs, images were processed by flattening every pixel into a long vector and feeding it into a plain neural network (a fully-connected network). For a 28×28 grayscale image that's 784 inputs. For a 224×224 color image? 150,528 inputs per sample.

That explosion in parameters creates three problems:

Problem 1 — Scale: A single hidden layer with 1000 neurons on a 224×224 image needs 150 million weights. The model can't learn anything useful without billions of examples.
Problem 2 — No spatial awareness: Shuffling all the pixels randomly would give the same flat vector to the network. It has no idea that nearby pixels are related.
Problem 3 — No translation invariance: A "7" in the top-left corner looks completely different from a "7" in the bottom-right, as far as the flat network is concerned. You'd need separate copies of every feature detector for every location.

CNNs solve all three problems with one elegant idea: shared, local filters. Instead of every neuron connecting to every pixel, a small filter (say, 3×3) slides across the image. The same filter weights are reused at every location. This is the convolution operation.

Section III

The Convolution Operation

A convolution is beautifully simple: take a small grid of numbers called a kernel (or filter), slide it over the input image one position at a time, and at each position compute the dot product — multiply corresponding values and sum the results. This produces a single output number per position, and the collection of all output numbers is the feature map.

Feature_Map[i, j] = Σ Σ Input[i+m, j+n] × Kernel[m, n]
↑ sum over all kernel positions (m, n)

Output size = (Input size − Kernel size + 2×Padding) / Stride + 1

The key insight: the kernel is the same everywhere. It learns one pattern (say, a vertical edge), and it looks for that pattern at every location in the image simultaneously. This is called parameter sharing, and it's the secret sauce.

Section IV

Feature Maps — What Do Kernels Detect?

A single kernel produces one feature map — one channel of learned responses. But in practice, a layer uses many kernels simultaneously. If layer 1 has 32 kernels, it produces 32 feature maps. Early layers learn low-level features; deeper layers combine those into complex patterns.

In a trained CNN, you don't choose what the kernels detect. The network learns them through backpropagation. Early layers reliably learn edge detectors, color blobs, and frequency detectors — very similar across different architectures and tasks. Deeper layers combine these primitives into increasingly complex patterns: textures, object parts, entire objects.

Section V

ReLU — The Non-linearity That Unlocks Power

After each convolution, we apply an activation function. Without it, stacking layers would be useless — any series of linear operations collapses into a single linear operation, no matter how deep.

The dominant choice is ReLU (Rectified Linear Unit):

ReLU(x) = max(0, x)
Simply: if negative → zero. If positive → keep it.

This is almost laughably simple, yet incredibly effective. Negative values (the kernel didn't find the pattern here) are killed. Positive values (the kernel did find it) pass through unchanged. It's a detector: "yes I see it" vs "not here."

The left shows a feature map with both positive (the kernel fired) and negative (no match) values, represented as light and dark. ReLU zeros out all negatives — the right shows only where the kernel detected something.

Why not sigmoid or tanh? Older activations like sigmoid saturate (gradients vanish to near-zero for large inputs). ReLU avoids this — its gradient is always 0 or 1 for active neurons. This simple property is why deep networks became trainable.
Section VI

Pooling — Compressing the Map

After convolution + ReLU, feature maps are still large. Pooling downsamples them — reducing spatial size while keeping the essential information. The most common variant is Max Pooling: slide a 2×2 window across the map, keep only the maximum value.

Max pooling achieves two things: it reduces computation for subsequent layers, and it builds in a small amount of translation invariance — a feature detected slightly to the left or right will still produce the same max output.

Section VII

Stacking Layers — Building Depth

The power of CNNs comes from stacking these operations. A single convolution finds simple local patterns. Stack two convolutions and the second one sees patterns of patterns. The effective receptive field — the region of the original image each neuron can see — grows with every layer.

With 3×3 kernels and no pooling: Layer 1 sees 3×3. Layer 2 sees 5×5. Layer 5 sees 11×11 of the original image. Add pooling and the receptive field grows even faster.

This hierarchical composition is what makes CNNs so powerful: simple primitives (edges → textures → parts → objects) assembled automatically through learning, without anyone writing the rules.

Section VIII

The Full CNN Architecture

A complete CNN for image classification typically has two parts: a convolutional backbone (Conv + ReLU + Pool, repeated) that extracts features, and a classifier head (fully-connected layers) that maps those features to class probabilities.

Section IX

Training — Learning the Kernels

The kernels aren't designed by hand — they're learned. Training works exactly the same as any neural network: forward pass, compute loss, backpropagate gradients, update weights. The magic is that gradients flow backward through the convolution operation to adjust each kernel value.

Loss (Cross-Entropy) = −Σ y_i · log(ŷ_i)
↑ Penalize confident wrong predictions

Gradient descent: kernel ← kernel − lr · ∂Loss/∂kernel
↑ Nudge each kernel weight to reduce the loss

One beautifully surprising fact: the gradient of a convolution is also a convolution. Backprop through a conv layer is computationally the same operation — just with a flipped kernel. This symmetry makes CNNs highly efficient to train on GPUs.

During training, a few practical techniques are essential:

Mini-batches: Process 32–256 images at once instead of one. The gradient is noisier but much faster — and the noise helps escape local minima.

Data augmentation: Randomly flip, rotate, and crop training images. The same physical cat in different orientations forces the network to learn truly invariant features.

Dropout: Randomly zero out neurons during training. This prevents the network from memorizing the training set and forces robust, distributed representations.

Section X

Modern CNN Architectures

From LeNet's humble 61K parameters, CNNs have grown into architectures of remarkable sophistication. The core operations are the same — but the assembly is far more creative.

ArchitectureYearParamsKey Innovation
LeNet-51998~60KFirst practical CNN; proved the concept
AlexNet201260MReLU, dropout, GPU training; won ImageNet by landslide
VGGNet2014138MVery deep (16–19 layers), only 3×3 kernels throughout
GoogLeNet20147MInception modules: parallel multi-scale convolutions
ResNet-50201525MResidual connections: skip layers, train 152 layers deep
EfficientNet20195–66MCompound scaling: width, depth, and resolution together
ConvNeXt202229MCNN redesigned using transformer insights; still competitive
ResNet's key insight (2015): Instead of learning a transformation F(x), learn the residual F(x) + x. This tiny change made it possible to train networks 152 layers deep without gradients vanishing — solving what had seemed an insurmountable problem.
▶ The Residual Block — ResNet's Secret

The skip connection (identity shortcut) adds the input directly to the output. Gradients can flow backward through this path without passing through any weights — solving vanishing gradients for arbitrarily deep networks.

Today, the line between CNNs and Transformers is blurring. Vision Transformers (ViT) apply attention mechanisms to image patches and achieve state-of-the-art results. But CNNs remain dominant in embedded applications, real-time systems, and any setting where computational efficiency matters. The convolutional inductive bias — locality and translation invariance — is still a valuable prior for visual tasks.

Key takeaway: The fundamental operations you've learned here — convolution, ReLU, pooling, stacking — are the building blocks of every computer vision system in production today. From your phone's face unlock to self-driving cars to medical image diagnosis.
